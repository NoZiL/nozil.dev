# Site Concept

## Vision

A minimal, fast, opinionated personal site that reads like a product — not a template.
It should feel like Nicolas built it himself: clean engineering, good typography, no fluff.

## Pages

| Route        | Title      | Purpose                                         |
| ------------ | ---------- | ----------------------------------------------- |
| `/`          | Home / Bio | One-screen intro: who, what, links              |
| `/work`      | Work & CV  | Experience timeline + skills + CV download      |
| `/portfolio` | Portfolio  | Curated project showcase (cards → detail pages) |
| `/contact`   | Contact    | Short form + direct email fallback              |

## Navigation

- Sticky minimal top nav: logo/name · Work · Portfolio · Contact
- Footer: GitHub · LinkedIn · email · "built with Astro & Cloudflare"

## Cross-site Backlinks

- Header/footer links to `github.com/nozil` and LinkedIn `/in/nicolaszilli`
- GitHub profile README already links back to `nozil.dev`
- Meta tags (`rel="me"`) for identity verification on both ends

## Tone

- First person, direct, no buzzword soup
- Technical credibility shown through portfolio detail, not adjectives
- Warm but not casual — the site of someone who works closely with founders

## Internationalisation (i18n)

The site ships in two languages — **French (fr)** and **English (en, default)**.

| Locale | Route prefix             | Audience                                                   |
| ------ | ------------------------ | ---------------------------------------------------------- |
| `en`   | `/` (default, no prefix) | International recruiters, global dev community             |
| `fr`   | `/fr/`                   | French clients, local recruiters, French-speaking founders |

Implementation via Astro's built-in i18n routing (`i18n.defaultLocale = 'en'`).
Content strings live in `src/i18n/en.ts` and `src/i18n/fr.ts`.
Long-form content (bio paragraphs, project descriptions) uses locale-specific `.md` files in each collection entry.

The language toggle sits in the nav (flag or `EN / FR` text switch). `<link rel="alternate" hreflang>` tags on every page for search engine locale signals.

**Phase plan**: ship English first, layer in French once content is locked.

## Performance Goals

- Lighthouse score ≥ 95 across all categories
- Zero client-side JS on bio and CV pages (Astro islands only where needed)
- Sub-1s LCP on mobile

## SEO & LLM Discoverability

Goal: appear in Google/Bing organic results _and_ be cited by LLMs (ChatGPT, Claude, Perplexity) when someone searches for freelance React Native developers in France.

### Technical SEO

- `sitemap.xml` generated by `@astrojs/sitemap` (includes `hreflang` alternates)
- `robots.txt` — allow all crawlers, explicitly allow AI agents (GPTBot, ClaudeBot, PerplexityBot)
- Canonical tags on every page
- `og:*` and `twitter:card` meta on every page
- JSON-LD structured data: `Person` schema on home, `ItemList` on portfolio, `BreadcrumbList` on all pages

### LLM Discoverability

- `llms.txt` at the root — machine-readable plain-text summary of who Nicolas is, what he does, and how to contact him. See [llmstxt.org](https://llmstxt.org) spec.
- `llms-full.txt` — extended version with CV and portfolio data in plain text
- Prose content written in clear, factual language (LLMs weight semantic clarity)
- GitHub README and LinkedIn bio backlink to nozil.dev — distributed authority signals

### Content SEO

- Each portfolio project page targets specific long-tail keywords (e.g. "React Native EHS app case study")
- `/work` page structured as a readable CV — LLMs extract this when asked about Nicolas
- Blog/notes section (future, not MVP) for topical authority on React Native and mobile engineering
